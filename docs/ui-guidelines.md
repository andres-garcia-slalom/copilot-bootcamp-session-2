# UI Guidelines

The TODO app should follow these core UI guidelines:

1. Use a clean, modern, and minimal visual design that keeps the task list easy to scan.
2. Use Material Design-inspired components for inputs, buttons, cards, dialogs, and navigation patterns.
3. Apply a consistent color palette with clear contrast between primary actions, secondary actions, completed tasks, and overdue tasks.
4. Make primary actions, such as adding or saving a task, visually prominent with consistent button styling.
5. Use clear visual indicators for task states, including active, completed, and overdue.
6. Ensure the layout is responsive and works well on desktop, tablet, and mobile screen sizes.
7. Keep forms simple and easy to complete, with clear labels and inline validation when needed.
8. Support accessibility best practices, including keyboard navigation, sufficient color contrast, and semantic labels for interactive elements.
9. Use readable typography and spacing that make the interface uncluttered and easy to understand.
10. Provide user feedback for important actions, such as creating, editing, deleting, or completing a task.

## Intent

The app should feel simple, polished, and accessible, helping users manage tasks efficiently without visual clutter or confusing interactions.
