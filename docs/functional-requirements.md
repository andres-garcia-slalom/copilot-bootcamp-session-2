# Functional Requirements

## Core TODO App Requirements

1. The user can create a new task by entering a title or description.
2. The user can edit an existing task's title or description after it has been created.
3. The user can delete a task that is no longer needed.
4. The user can mark a task as complete.
5. The user can mark a completed task as incomplete.
6. The user can assign a due date to a task.
7. The user can view all tasks in a single list.
8. The user can view completed tasks separately from incomplete tasks.
9. The tasks are sorted by due date, with the earliest due date shown first.
10. Tasks without a due date are shown after tasks that have a due date.
11. The user can see when a task is overdue.
12. The user can save tasks so they remain available after refreshing or reopening the app.
13. The user can add multiple tasks without losing existing tasks.
14. The user can clearly distinguish each task's title, completion status, and due date.
15. The app prevents creation of a task with an empty title.
