# Functional Requirements

The TODO app should support the following core functional requirements:

1. Users can create a new task with a clear title/description.
2. Users can edit an existing task after it has been created.
3. Users can delete a task they no longer need.
4. Users can mark a task as complete or incomplete.
5. Users can assign an optional due date to a task.
6. Users can view all tasks in a single list.
7. Users can sort tasks by due date, completion status, or creation order.
8. Users can filter tasks to view all, active, or completed items.
9. Users can persist tasks so they remain available after refreshing or reopening the app.
10. Users can clearly distinguish overdue tasks from upcoming tasks.

## Intent

The app should provide a simple and intuitive way for users to manage personal tasks, track deadlines, and organize work by status and urgency.
